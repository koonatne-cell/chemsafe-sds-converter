# ChemSafe – แปลง SDS เป็นฟอร์มติดหน้างาน

เว็บแอปสำหรับ Safety Officer ในโรงงาน: อัปโหลดไฟล์ SDS (Safety Data Sheet, PDF ภาษาอังกฤษ)
ระบบดึงข้อมูลสำคัญออกมาให้อัตโนมัติ แปลเป็นไทย ให้ตรวจ/แก้ไข/แนบรูปภาพ
แล้ววางลงฟอร์ม "SDS ติดหน้างาน" เป็น PDF ที่พร้อมดาวน์โหลดไปปริ้นติดหน้างานได้ทันที

## สิ่งที่ต้องมีก่อนเริ่ม

- Python 3.10 ขึ้นไป (เครื่องนี้ติดตั้ง Python 3.12 ให้แล้ว)
- ไม่จำกัดระบบปฏิบัติการ (ฟอนต์ไทย Noto Sans Thai แนบมากับโปรเจกต์แล้วใน `assets/fonts/` รันได้ทั้ง Windows/Linux/Mac)

## วิธีติดตั้ง (ทำครั้งเดียว)

1. เปิด Command Prompt หรือ PowerShell ที่โฟลเดอร์นี้
2. ติดตั้ง library ที่จำเป็นด้วยคำสั่ง:

   ```
   pip install -r requirements.txt
   ```

3. คัดลอกไฟล์ `.env.example` เป็น `.env` แล้วตั้งค่า:
   - `ADMIN_PASSCODE` = รหัสผู้ดูแลที่คุณต้องการ (ค่าเริ่มต้นตอนนี้คือ `admin1234` ในไฟล์ `.env` ที่สร้างไว้ให้แล้ว **ควรเปลี่ยนก่อนใช้งานจริง**)
   - `SECRET_KEY` = ข้อความสุ่มยาวๆ อะไรก็ได้ (ใช้เข้ารหัส session)
   - `GOOGLE_TRANSLATE_API_KEY` = ปล่อยว่างไว้ได้ ถ้ายังไม่มี (ระบบจะใช้ตัวแปลฟรีแทนอัตโนมัติ)

## วิธีรัน

```
uvicorn main:app --reload
```

แล้วเปิดเบราว์เซอร์ไปที่ `http://127.0.0.1:8000`

## วิธีใช้งาน (พนักงาน)

1. (ไม่บังคับ) กรอกช่อง "ข้อมูลผู้จัดทำ / ผู้อนุมัติ" ด้านบนสุด (ชื่อ-สกุล + ตำแหน่ง ของทั้งสองฝั่ง)
   ค่าพวกนี้จะไปวางท้ายฟอร์ม PDF ตรงบล็อก "จัดทำโดย / อนุมัติโดย"
2. กด "เลือกไฟล์ SDS" แล้วเลือกไฟล์ PDF จากนั้นกด "ดึงข้อมูลจาก SDS"
3. ตรวจ/แก้ไขข้อมูลในแต่ละช่อง (สำคัญมาก เพราะเป็นงานความปลอดภัย) รวมถึงช่อง "ชื่อสารเคมี (แถบเหลือง)"
   และ "Signal Word (แถบแดง)" ที่อยู่บนหัวฟอร์ม (ปกติ SDS ไม่ค่อยระบุ Signal Word ตรงๆ ต้องเติมเองส่วนใหญ่)
4. กด "แปลเป็นไทย" แล้วตรวจคำแปลอีกครั้ง (แก้ได้ตามต้องการ)
5. ตรวจช่อง "สัญลักษณ์ GHS" ท้ายฟอร์ม ระบบจะติ๊กให้อัตโนมัติเท่าที่เดาได้จากข้อความใน SDS
   (ดูจาก Section 2 เป็นหลัก) แต่การเดาไม่แม่นยำ 100% ต้องตรวจ/ติ๊กเพิ่ม-ลบเองอีกครั้งเสมอ
   ก่อนสร้าง PDF (สัญลักษณ์ที่ติ๊กจะไปวางในกล่องสี่เหลี่ยมมุมซ้ายบนของฟอร์ม)
6. แนบรูปสลากสารเคมี และรูปภาชนะบรรจุ (ถ้ามี)
7. กด "สร้าง PDF" แล้วกด "ดาวน์โหลด PDF" เพื่อบันทึกลงเครื่องและปริ้น

## สร้างฉลากภาชนะบรรจุสารเคมี (หน้าแยก /label)

ไปที่ `http://127.0.0.1:8000/label` (หรือกดลิงก์ "สร้างฉลากภาชนะบรรจุ" ที่หัวหน้าเว็บหลัก)
เป็นหน้าแยกต่างหากสำหรับสร้างฉลากติดภาชนะ/ขวดสารเคมี (secondary container label) ตามหัวข้อที่
มาตรฐาน GHS กำหนด (ชื่อผลิตภัณฑ์, สัญลักษณ์อันตราย, คำสัญญาณ, Hazard/Precautionary statement,
ข้อมูลผู้ผลิต) ไม่ใช่ฟอร์ม "SDS ติดหน้างาน" แบบหน้าแรก

1. อัปโหลดไฟล์ SDS แล้วกด "ดึงข้อมูลจาก SDS" (ใช้ตัวดึงข้อมูลเดียวกับหน้าแรก)
   ระบบจะลองดึง Hazard statement (H-code) และ Precautionary statement (P-code) จาก Section 2
   ให้อัตโนมัติเป็นตัวอย่างแรกก่อน **แต่ SDS หลายฉบับไม่ได้ระบุโค้ดเหล่านี้ตรงๆ (เขียนบรรยายเป็นประโยคเฉยๆ
   หรือใช้คำว่า "Void" เพราะสารนั้นไม่เข้าเกณฑ์ต้องติดป้าย)** ถ้าดึงมาว่างเปล่าหรือไม่ครบ ต้องพิมพ์เพิ่ม/แก้เอง
   ในกล่องข้อความ (บรรทัดละ 1 ข้อ) เสมอ ห้ามปล่อยว่างถ้าสารนั้นมีอันตรายจริง
2. ตรวจ/แก้ไขทุกช่อง ติ๊กสัญลักษณ์ GHS ให้ตรง (ระบบติ๊กให้อัตโนมัติเท่าที่เดาได้เหมือนหน้าแรก)
3. เลือกขนาดฉลาก (เล็ก 70x100มม. / กลาง 100x150มม. / ใหญ่ 150x200มม. — ดูปรับใน `fields.py`
   ตัวแปร `LABEL_SIZE_PRESETS`) แล้วกด "สร้าง PDF ฉลาก"
4. ไฟล์ PDF ที่ได้ **มีขนาดหน้ากระดาษเท่าฉลากจริงเป๊ะ ไม่ใช่ A4** (ใช้ `reportlab` ตั้งขนาดหน้าตรงๆ)
   ถ้าพิมพ์ด้วยเครื่องพิมพ์สติกเกอร์ที่ตัดกระดาษตามขนาดจะได้ขนาดพอดีทันที แต่ถ้าพิมพ์ด้วยเครื่องพิมพ์ทั่วไป
   ที่ใส่กระดาษ A4 **ต้องตั้ง dialog เครื่องพิมพ์เป็น "Actual size / 100%"** ไม่ใช่ "Fit to page"
   ไม่งั้นเครื่องพิมพ์จะขยายฉลากให้เต็ม A4 ให้เอง

หน้านี้เป็นเวอร์ชัน function-first (เน้นให้ใช้งานได้ครบตามข้อกำหนด GHS ก่อน) ยังไม่ได้ปรับ UX/พรีวิว
ให้สวยงามเท่าหน้าแรก โค้ดหลักอยู่ที่ `label_template.py` (วาด PDF ขึ้นใหม่ทั้งหมด ไม่ได้ใช้ Template.pdf
เดิมเพราะเป็น layout คนละแบบ) และ `templates/label.html` + `static/label.js`

## วิธีใช้งาน (ผู้ดูแล / Admin)

1. ไปที่ `http://127.0.0.1:8000/admin`
2. กรอกรหัสผู้ดูแล (ค่าที่ตั้งไว้ใน `.env` ช่อง `ADMIN_PASSCODE`)
3. ดูรายการทะเบียนสารเคมีที่เคยสร้าง ดาวน์โหลดไฟล์ PDF ซ้ำได้ หรือลบรายการที่ไม่ต้องการ
4. อัปโหลด Template.pdf ใหม่มาแทนที่ไฟล์เดิมได้ (ดูหัวข้อ "เปลี่ยน Template" ด้านล่าง)

## เปลี่ยน Template ในอนาคต

หน้า Admin มีปุ่ม "อัปโหลด Template ใหม่" ใช้แทนที่ `assets/Template.pdf` ได้ทันที
(ระบบสำรองไฟล์เดิมไว้ให้อัตโนมัติที่ `assets/template_backups/` ทุกครั้งก่อนแทนที่)

**ข้อจำกัดสำคัญ:** วิธีนี้ใช้ได้เฉพาะไฟล์ที่ตำแหน่งช่องข้อมูลเหมือนเทมเพลตเดิมทุกจุด
(เช่น เปลี่ยนแค่ชื่อบริษัท โลโก้ เลขที่เอกสาร วันที่) เพราะระบบวางข้อความด้วยพิกัด (x, y) ที่ตายตัว
ที่กำหนดไว้ใน `fill_template.py` (dict ชื่อ `HEADER`, `SHORT`, `BLOCK`, `NFPA`, `SIGNATURE`, `IMAGE_BOXES`)
ถ้าอัปโหลดเทมเพลตที่ตำแหน่งช่องต่างจากเดิมจริงๆ (เช่น ขนาดกระดาษเปลี่ยน, จัดหน้าใหม่) ข้อมูลจะไปวางผิดที่
กรณีนี้ต้องแก้ไขพิกัดในไฟล์ `fill_template.py` เอง (ใช้ `pdfplumber` หาตำแหน่งข้อความ/เส้นกรอบในเทมเพลตใหม่)

## โครงสร้างไฟล์

```
main.py               หน้าเว็บ + API ทั้งหมด (FastAPI) - จุดเริ่มรัน uvicorn main:app อยู่ที่รากโปรเจกต์เท่านั้น

core/                 โมดูลตรรกะหลัก (ไม่เกี่ยวกับการวาด PDF)
  parser.py             ดึงข้อมูลจาก SDS (PDF) ด้วย regex (ลองหลาย pattern ต่อฟิลด์)
  translator.py         แปลอังกฤษ -> ไทย
  database.py           จัดการฐานข้อมูล SQLite (ทะเบียนสารเคมี)
  auth.py               เช็ครหัสผู้ดูแล
  fields.py             รายชื่อฟิลด์ทั้งหมด จัดกลุ่ม+ป้ายภาษาไทย (รวมช่องจัดทำโดย, สัญลักษณ์ GHS,
                         ขนาดฉลากที่เลือกได้ใน LABEL_SIZE_PRESETS, ชื่อผู้อนุมัติคงที่ใน APPROVED_NAME)

pdf_gen/               โมดูลวาด/สร้างไฟล์ PDF (และ Excel เดิม)
  fill_template.py      วางข้อมูล+รูปภาพลงบน Template.pdf (ฝังฟอนต์ไทย) ใช้งานจริงตอนนี้
  label_template.py     วาดฉลากภาชนะบรรจุสารเคมีขึ้นใหม่ (ไม่ใช้ Template.pdf) หน้า /label
  fill_excel.py          -- เคยลองสลับไปใช้ Excel เป็นเอาต์พุตแทน PDF แต่กลับมาใช้ PDF เหมือนเดิม
                         เก็บโค้ดไว้เผื่ออยากกลับไปใช้ในอนาคต ไม่ได้เรียกจากหน้าเว็บแล้ว --

templates/            หน้าเว็บ (HTML) - index.html (SDS ติดหน้างาน), label.html (ฉลากภาชนะบรรจุ)
static/               CSS + JavaScript (app.js = หน้าแรก, label.js = หน้าฉลาก)
static/ghs_icons/         ไอคอนสัญลักษณ์ GHS 9 ชนิด (สำเนาไว้ให้หน้าเว็บโหลดแสดงเป็น checkbox)
assets/Template.pdf       แม่แบบฟอร์ม SDS ติดหน้างาน (ห้ามลบ)
assets/template_backups/  ไฟล์ Template.pdf เก่าที่สำรองไว้ตอน Admin อัปโหลดตัวใหม่
assets/fonts/             ฟอนต์ไทยที่ฝังใน PDF (Noto Sans Thai, ฟรี, แจกจ่ายต่อได้ตามกฎหมาย)
assets/ghs_icons/         ไอคอนสัญลักษณ์ GHS 9 ชนิด ต้นฉบับ (ใช้วาดลง PDF จริงตอนสร้าง)
assets/Template.xlsx      แม่แบบฟอร์มแบบ Excel (ตัดมาจากไฟล์ต้นแบบจริงของบริษัท ใช้กับ pdf_gen/fill_excel.py)
data/                 ฐานข้อมูล + ไฟล์ที่ผู้ใช้อัปโหลด/สร้าง (สร้างอัตโนมัติตอนรันครั้งแรก)
```

**หมายเหตุ:** `core/` และ `pdf_gen/` เป็น Python package (มี `__init__.py`) รันไฟล์เดี่ยวๆ ข้างในเพื่อทดสอบ
ต้องรันจากรากโปรเจกต์ด้วย `python -m` เท่านั้น (เช่น `python -m pdf_gen.fill_template`) ไม่ใช่รันตรงๆ
ด้วย path (เช่น `python pdf_gen/fill_template.py`) เพราะ import ข้ามโฟลเดอร์ (เช่น `from core.parser import ...`)
ต้องอาศัยการรันแบบ package ถึงจะ resolve ได้ถูกต้อง

## หมายเหตุสำคัญ

- ฟิลด์ไหนที่ระบบดึงจาก SDS ไม่ได้ จะใส่ "-" ให้แทน ไม่ทำให้แอปพัง แต่ต้องเติมเองก่อนสร้าง PDF
- ทุกช่องข้อมูลแก้ไขได้เสมอก่อนสร้าง PDF เพราะเป็นงานความปลอดภัย ต้องมีคนตรวจยืนยันทุกครั้ง
- SDS ส่วนใหญ่ต้องเป็นไฟล์ PDF ที่มีตัวอักษรจริง (ไม่ใช่รูปสแกน) ถ้าเป็นรูปสแกนล้วนๆ ระบบดึงข้อความไม่ได้
- **รองรับ SDS ภาษาไทยโดยตรงด้วย** ไม่ต้องเป็นภาษาอังกฤษเสมอไป ถ้า SDS ต้นฉบับเป็นภาษาไทยอยู่แล้ว ระบบจะดึงข้อมูลจาก
  ป้ายภาษาไทย (เช่น "ชื่อทางการค้า", "ทางตา/ทางผิวหนัง/ทางปาก/ทางการหายใจ", "การดับเพลิง", "วิธีปฏิบัติเมื่อมีการหกรั่วไหล",
  "การเก็บรักษา", "วิธีการกำจัด") และหัวข้อ "ส่วนที่ N" ได้เลยโดยไม่ต้องผ่านขั้นตอนแปลภาษา (ฟิลด์ที่ดึงได้ตรงๆ แบบนี้
  จะไม่ถูกส่งเข้า Google Translate ซ้ำ) ถ้า SDS ผสมสองภาษาในไฟล์เดียว ระบบจะลองทั้งป้ายอังกฤษและป้ายไทยกับทุกฟิลด์
  ใครเจอ SDS ภาษาไทยที่ดึงไม่ครบ ให้เปิดไฟล์ดูป้ายที่ใช้จริงแล้วเพิ่ม pattern ภาษาไทยใน `parser.py` เพิ่มได้เช่นกัน
- **การดึงข้อมูลใช้ regex หลายรูปแบบต่อฟิลด์** (ดูใน `parser.py`) เพื่อให้ครอบคลุม SDS จากหลายผู้ผลิตมากขึ้น
  แต่ก็ยังไม่ครอบคลุมทุกรูปแบบในโลก ถ้าเจอ SDS ที่ดึงไม่ได้เลย (ทุกช่องเป็น "-") ให้เปิดไฟล์ดูข้อความ Section
  ที่เกี่ยวข้องแล้วเพิ่ม pattern ใหม่ใน `parser.py` (ฟังก์ชัน `grab()` รับ list ของ pattern ได้อยู่แล้ว)
- ฟิลด์ Section 4 (การปฐมพยาบาล: `fa_*`) กับ Section 11 (พิษวิทยา: `hz_*`) บาง SDS ใช้ป้ายย่อยชื่อเดียวกัน
  (เช่น "Eyes:", "Skin:", "Ingestion:", "Inhalation:") ทั้งสอง section แต่ความหมายคนละเรื่อง ระบบเลยตัดข้อความ
  มาเฉพาะช่วง "SECTION 4" / "SECTION 11" ก่อนค้นหา (ฟังก์ชัน `extract_section()` / `grab_scoped()`) กันจับผิด section
  ถ้า SDS ไม่มีหัว "SECTION N" ชัดเจน ระบบจะ fallback ไปค้นทั้งไฟล์แทน
- ดัชนี NFPA มักอยู่ในรูปไดอะแกรมเพชร ไม่ใช่ข้อความเรียงปกติ ดึงยากและเสี่ยงไปจับตัวเลขผิดจุด (เช่น ปนกับตาราง HMIS
  ที่มักอยู่ใกล้กัน) ระบบเลยจำกัดให้ค้นเฉพาะในช่วงข้อความถัดจากคำว่า "NFPA" ก่อน (ฟังก์ชัน `grab_near()`) และรับ
  เฉพาะเลข 0-4 เท่านั้น **ถึงอย่างนั้นก็ยังไม่การันตีถูก 100%** เพราะโครงสร้างไดอะแกรมแต่ละ SDS ไม่เหมือนกัน
  ต้องตรวจตัวเลข NFPA ในฟอร์มเทียบกับรูปไดอะแกรมจริงใน SDS ทุกครั้งก่อนสร้าง PDF
- ตอนนี้แปลไทยด้วยไลบรารีฟรี ถ้าอยากได้ผลแปลแม่นยำขึ้น ใส่ `GOOGLE_TRANSLATE_API_KEY` ใน `.env` ได้ทันทีโดยไม่ต้องแก้โค้ด (แก้ `translator.py` ให้เรียก `translate_with_google_cloud()` แทน)
- ฟอนต์ไทยที่ใช้คือ Noto Sans Thai (สัญญาอนุญาต SIL Open Font License, ฟรีและแจกจ่ายต่อในโค้ด public ได้) แนบไฟล์ `.ttf` มาในโปรเจกต์แล้วที่ `assets/fonts/` ไม่ต้องพึ่งฟอนต์จากเครื่อง/OS เลย รันได้ทั้ง Windows/Linux/Mac
  (ห้ามใช้ Tahoma หรือฟอนต์ลิขสิทธิ์ของ Microsoft/บริษัทอื่นแทน ถ้า repo เป็น public เพราะผิดสัญญาอนุญาตของฟอนต์นั้นๆ)
- ระบบตัดบรรทัดข้อความไทยด้วย `pythainlp` (กันตัดกลางคำ เช่น "นาที" ไม่กลายเป็น "นา" + "ที") และคำนวณพื้นที่ว่างของแต่ละช่องอัตโนมัติ ถ้าข้อความยาวมากจนไม่พอแม้ลดฟอนต์แล้ว ระบบจะตัดท้ายด้วย "…" แทนการปล่อยให้ล้นไปทับช่องถัดไป (ต้องตรวจข้อความในฟอร์มก่อนสร้าง PDF เสมอ ถ้าเห็น "…" แปลว่าเนื้อหาเดิมยาวเกินช่อง ควรแก้ให้สั้นลง)
- ช่อง "ชื่อสารเคมี" (แถบเหลือง) และ "Signal Word" (แถบแดง) บนหัวฟอร์ม วางกึ่งกลางแถบ พร้อมลดขนาดฟอนต์อัตโนมัติถ้าข้อความยาวเกิน (เหมือนช่องอื่นๆ)
- **สัญลักษณ์ GHS ทั้ง 9 ชนิด** ระบบเดาให้อัตโนมัติจากคำ/รหัส GHS ที่เจอใน SDS (ฟังก์ชัน `detect_pictograms()` ใน `parser.py`)
  แต่เป็นการเดาแบบจับคำสำคัญล้วนๆ (ไม่เข้าใจบริบทประโยค) อาจ false positive ได้ เช่น เจอคำว่า "toxic" ในประโยคปฏิเสธ
  ("not toxic") ก็ยังติ๊กให้เหมือนกัน **ต้องตรวจ/แก้ไขเองในฟอร์มทุกครั้งก่อนสร้าง PDF** สัญลักษณ์ที่ติ๊กไว้จะถูกจัดวาง
  เป็นตาราง (grid) ในกล่องสี่เหลี่ยมมุมซ้ายบนของฟอร์มโดยอัตโนมัติ (1 อัน = เต็มกล่อง, 2-4 อัน = ตาราง 2x2, มากกว่านั้น = 3 คอลัมน์)
- **ค่าที่บอกว่า "ไม่มีข้อมูล"** เช่น "No data available", "Not applicable", "No special...", "None" ระบบจะดึงมาเป็น
  "-" ให้อัตโนมัติ (ไม่ก็อปข้อความ "No data available" มาใส่ในฟอร์มตรงๆ) ดูรายการทั้งหมดได้ที่ `_NO_DATA_PREFIXES`
  ใน `parser.py` ถ้าเจอวลีอื่นที่ควรนับเป็น "ไม่มีข้อมูล" เพิ่มเข้าไปใน list นี้ได้เลย
- **SDS หลายฉบับจัดหน้าเป็นตาราง 2 คอลัมน์** (เช่น "Product AS SOLD" คู่กับ "Product AT USE DILUTION") ระบบจะดึงมา
  เฉพาะคอลัมน์แรกเท่านั้น (หยุดจับที่ช่องว่างยาว 2 ตัวขึ้นไป ซึ่งมักหมายถึงคอลัมน์ถัดไป) ถ้าอยากได้ค่าจากคอลัมน์อื่น
  ต้องแก้เองในฟอร์มหลังดึงข้อมูล
- **ค่าที่ยาวจน wrap หลายบรรทัดใน SDS** (เช่น วิธีกำจัดของเสียที่มีหลายประโยคต่อกันหลายบรรทัด) ระบบจะรวมมาเป็นค่าเดียว
  ให้อัตโนมัติ (ฟังก์ชัน `block()` ใน `parser.py` หยุดจับเมื่อเจอบรรทัดว่าง, บรรทัดที่ขึ้นต้นด้วยสัญลักษณ์บูลเล็ต
  "·"/"•", หรือบรรทัดที่ดูเหมือนหัวข้อใหม่) ถ้าเจอ SDS รูปแบบที่ยังตัดข้อความสั้นเกินไปหรือยาวเกินไป ให้ปรับ
  `_BLOCK_END` ในไฟล์เดียวกัน
- ช่อง pH ตอนนี้แสดงผลได้แล้ว (เดิมไม่มีพิกัดจึงหายไปเงียบๆ) ต้องปิดขีด "-" ตัวอย่างของเทมเพลตก่อนวางค่าจริงเสมอ (ดู `SHORT_COVER` ใน `fill_template.py`)
- **CAS No. เมื่อ Section 3 มีหลายสารเคมี (ผลิตภัณฑ์เป็นส่วนผสม)** ระบบจะเลือกเลข CAS ของสารที่มี Concentration (%)
  สูงสุดให้อัตโนมัติ (ฟังก์ชัน `parse_cas_from_composition_table()` ใน `parser.py` อ่านค่าบนสุดของช่วง % เช่น
  "30 - 60" ใช้ 60) ถ้าหาตารางแบบนี้ไม่เจอ (สารเดี่ยว) จะ fallback ไปหา "CAS No:" แบบปกติแทน
- **ช่องขาว (Special Hazard) ในรูปเพชร NFPA** เป็น dropdown เลือกได้ทีละ 1 ค่า (OXY/ACID/ALK/COR/W หรือไม่มี)
  ระบบเดาให้เบื้องต้นจากสัญลักษณ์ GHS ที่ตรวจเจอ (oxidizer -> OXY, corrosive -> COR) แต่ SDS ไม่ได้ระบุค่านี้ตรงๆ
  ต้องตรวจสอบเองเสมอ ตัวหนังสือวางกึ่งกลางช่องขาวด้านล่างของรูปเพชร (พิกัด `NFPA_SPECIAL_X_CENTER`/`NFPA_SPECIAL_TOP_CENTER` ใน `fill_template.py`)

## Deploy ขึ้นเว็บจริง (GitHub + Render)

### 1. อัพโค้ดขึ้น GitHub

โฟลเดอร์นี้เตรียม git ไว้ให้แล้ว (`.gitignore` กันไฟล์ลับ/ข้อมูลรันไทม์ไม่ให้หลุดขึ้นไป) ที่เหลือแค่สร้าง repo
บน GitHub แล้ว push (เครื่องนี้ไม่มี `gh` CLI ติดตั้งให้ ต้องทำตามขั้นตอนนี้เอง):

1. ไปที่ [github.com/new](https://github.com/new) สร้าง repository ใหม่ (**อย่าติ๊ก** "Add a README" เพราะมีอยู่แล้ว)
2. รันคำสั่งนี้ในโฟลเดอร์โปรเจกต์ (แทน `<your-username>` และ `<repo-name>` ด้วยของจริง):

   ```bash
   git remote add origin https://github.com/<your-username>/<repo-name>.git
   git branch -M main
   git push -u origin main
   ```

### 2. Deploy บน Render

1. ไปที่ [render.com](https://render.com) สมัคร/login ด้วยบัญชี GitHub
2. กด "New +" -> "Blueprint" แล้วเลือก repo นี้ (Render จะอ่านค่าตั้งต้นจาก `render.yaml` ที่เตรียมไว้ให้อัตโนมัติ)
3. ตอนสร้าง service ระบบจะถามหา environment variables ที่ทำเครื่องหมาย `sync: false` ไว้ ให้กรอก:
   - `ADMIN_PASSCODE` = รหัสผู้ดูแลจริง (**ห้ามใช้ `admin1234` ค่าทดสอบเดิม**)
   - `SECRET_KEY` = ข้อความสุ่มยาวๆ
   - `GOOGLE_TRANSLATE_API_KEY` = เว้นว่างได้ถ้ายังไม่มี
4. กด Deploy รอสักครู่ ได้ URL แบบ `https://chemsafe-xxxx.onrender.com` มาใช้งานได้ทันที (ทุกคนเข้าได้จากที่ไหนก็ได้ ไม่ต้องอยู่ WiFi วงเดียวกัน)

### ข้อจำกัดสำคัญของ Render free tier

- **ข้อมูลหายเมื่อ redeploy/restart เครื่อง** เพราะ free tier ไม่มี persistent disk ไฟล์ `data/chemsafe.db` (ทะเบียนสารเคมี)
  และรูปที่อัปโหลด/PDF ที่สร้างไว้จะหายไปทุกครั้งที่ deploy ใหม่หรือเครื่องรีสตาร์ท (Render free tier รีสตาร์ทอัตโนมัติ
  ถ้าไม่มีคนเข้าใช้นานเกิน 15 นาทีด้วย) เหมาะกับการ demo/ทดลองใช้ก่อน ถ้าจะใช้งานจริงจังต้องอัป plan เป็นแบบมี
  persistent disk (~$7/เดือนขึ้นไป) แล้วผูก disk เข้ากับโฟลเดอร์ `data/` ในหน้า Render dashboard
- **โหลดช้าตอนไม่มีคนใช้นานๆ** free tier จะ "หลับ" ถ้าไม่มีคนเข้า พอมีคนเข้าใหม่ต้องรอเครื่องตื่นสักครู่ (10-30 วินาที)
